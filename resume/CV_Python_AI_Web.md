- Интегрировал операторскую панель: зеркалирование сообщений, включение/выключение бота в чате, уведомления при передаче телефона.

# Кейсы для портфолио и собеседований

## 1. Red Button — AI-консультант с триажом и базой знаний

**Задача:** Анонимный канал поддержки 24/7 для людей с зависимостями и их близких — без ощущения «бота», с эскалацией к живому специалисту при риске.

**Что сделал:**
- Диалоговый pipeline: режим (сам/близкий) → триаж безопасности → 5–7 шагов GPT → предложение созвона.
- Гибридная классификация тем: alias-индекс + GPT JSON + keywords.
- RAG: в промпт подставляются фрагменты базы знаний по теме и режиму.
- Детекция risk-флагов (суицид, передоз, судороги) с приоритетом экстренных инструкций.

**Результат:** единый сценарий в Telegram и на сайте; персонализация по теме; операторы получают контекст до звонка.

---

## 2. Red Button — Backend и интеграции под продакшен

**Задача:** Стабильный сервис с историей диалогов, защитой от дублей webhook, передачей лидов специалистам и работой операторской панели.

**Что сделал:**
- PostgreSQL: users, sessions, messages, processed_updates (идемпотентность).
- API веб-чата с CORS; общий `handle_incoming_message` для Telegram и web.
- Интеграция с панелью операторов; фоновый экспорт завершённых диалогов.
- Деплой на Render, hot-reload config и knowledge base.

**Результат:** продакшен-сервис; операторы видят переписку в реальном времени; админы получают транскрипты без ручного экспорта.

---

## 3. Red Button — База знаний без разработчика

**Задача:** Специалисты добавляют сценарии поддержки без правки кода и деплоя.

**Что сделал:**
- GPT-парсер свободного текста в JSON-правило.
- Валидатор схемы и медицинских ограничений.
- Атомарная запись в KB с бэкапом и дедупликацией.

**Результат:** добавление знания через `/kb_add` в Telegram; обновление контента за минуты, не дни.
